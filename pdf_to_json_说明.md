# PDF 到 JSON 及图像处理工具说明 (pdf_to_json.py)

本工具是一个基于 Python 的 PDF 页面级处理脚本，主要利用 `PyMuPDF (fitz)` 和 `Pillow (PIL)` 库，实现将 PDF 文件按页拆解、提取带有丰富排版信息的文本区块、进行不同坐标系数据的映射合并，并支持图像渲染与坐标可视化标注。

## 核心功能

1. **PDF 页面排版数据提取 (`extract_pdf_to_json`)**
   - **逐页深度解析**: 使用 PyMuPDF 的 `dict` 模式提取包含字体、大小、位置、上标等丰富排版信息的文本。
   - **特殊字符与排版修复**: 自动处理脚注序号（如 ①、② 变 a、b）的识别错误，自动修补中英混排时丢失的空格。
   - **双轨输出**: 每页导出包含文本与精确 `bbox` 坐标的 JSON 文件（如 `1.json`）；同时以 2 倍缩放 (`fitz.Matrix(2, 2)`) 渲染并保存高清 PNG 页面图片。

2. **多数据源坐标可视化标注**
   - **PyMuPDF 坐标可视化 (`draw_boxes_on_images`)**: 读取上一步提取的 JSON，自动计算缩放比例，在渲染的 PNG 页面上绘制红色文本区块边界框。
   - **DS-OCR 坐标可视化 (`draw_boxes_on_images_ds_ocr`)**: 支持读取第三方（如 DS-OCR）的 JSON 格式（通常将原图宽度统一缩放至 1024），自动将坐标还原并映射到真实图片上绘制红框。

3. **双重 JSON 数据空间映射与合并 (`merge_py_json_to_ds_json`)**
   - **跨坐标系映射**: 将基于原始 PDF 尺寸的 PyMuPDF 文本坐标，等比例转换为 DS-OCR 格式（1024 基准）的坐标系。
   - **空间包含匹配**: 遍历 DS-OCR JSON 中的目标区域 `boxes`，提取 PyMuPDF JSON 中落在该区域内的文本内容。
   - **容错拼接**: 将匹配到的精准文本拼接后，以 `text_content` 字段的形式平级写入到目标 DS-OCR JSON 的对应区块节点中。

## 使用方法

### 1. 依赖安装
运行此脚本前需确保安装了核心依赖库：
```bash
pip install pymupdf pillow
```

### 2. 配置路径与执行
打开 `pdf_to_json.py`，在底部的 `if __name__ == "__main__":` 块中配置所需的输入输出目录：
- `pdf_file`: 输入的 PDF 文件路径
- `out_folder`: 存放 PyMuPDF 提取出 JSON 文件的目录
- `img_folder`: 存放提取出的高清页面图片的目录
- `ds_json_folder`: （可选）存放外部传入的 DS-OCR 格式 JSON 的目录
- `merged_output_folder`: 合并后生成的新 JSON 存放目录

配置完成后，在终端运行：
```bash
python pdf_to_json.py
```
*注：脚本中的四大步骤可根据实际需求通过注释/取消注释代码来按需独立执行。*

## 关键问题的处理说明与逻辑记录

在工具的开发过程中，针对复杂的 PDF 解析与多源数据整合场景，实现了以下关键逻辑：

### 1. 脚注图标被错误识别为英文字母（如 ① 变成 a）
**问题现象**: PDF 底层通常使用普通字母（a、b、c）结合特定的“符号字体”来实现带圈数字（①、②）的显示。PyMuPDF 默认提取底层字母，导致文本错乱。
**处理逻辑**:
- 使用 `page.get_text("dict")` 模式代替普通的纯文本提取，以获取每一个文本片段（`span`）的字体名称、字号大小和坐标。
- **启发式特征检测**: 当检测到提取内容为 `a-z` 字母时，综合判断以下特征：
  1. 是否设置了上标标识 (`flags & 1 != 0`)。
  2. 字号是否明显小于同行的最大字号，且垂直位置偏上（典型上标）。
  3. 是否处于页面底部的行首（典型脚注说明）。
  4. 字体名称是否包含特定的符号字体关键词（如 `symbol`, `wingding`, `dingbat`, `ropesequencenumber`）。
- 若命中上述特征，则通过内置的 `CIRCLE_MAP` 字典，将 `a-t` 映射转换回真实的 `①-⑳` 符号。

### 2. 跨格式系统的坐标系统一
**问题现象**: PyMuPDF 提取的坐标基于 PDF 文档的内部点(Point)尺寸（如 595x842），而外部模型（DS-OCR）输出的坐标基于将原图宽度缩放至 1024 后的相对尺寸。
**处理逻辑**:
- 在合并数据与绘制图片框时，通过读取目标 JSON 中的 `image_dims`（获取原图或目标基准的宽 `w` 和高 `h`）。
- 计算 X 轴和 Y 轴的独立缩放比例：`scale_x = ds_w / py_w`，`scale_y = ds_h / py_h`。
- 将源坐标按比例进行数学转换后再进行后续的绘制或空间匹配。

### 3. 区块重叠匹配与边界容错
**问题现象**: 在将 PyMuPDF 文本映射合并到 DS-OCR 提供的区块时，由于两者使用的分割算法不同，导致坐标区块无法绝对 100% 重合。
**处理逻辑**:
- **边界框外扩**: 在匹配前，将 DS-OCR 给定的目标边界框向四周**外扩 5 个像素** (`[x0-5, y0-5, x1+5, y1+5]`)，增加捕获面积。
- **中心点包含算法 (`is_contained_or_overlap`)**: 不要求矩形完全包含，只要 PyMuPDF 提取出的候选文本块的**中心点**落在外扩后的目标区域内，即视为匹配成功，从而大幅提高了容错率和合并的准确性。
